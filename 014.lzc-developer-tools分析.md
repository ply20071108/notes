## 3. 基础设施服务

### 3.1 SSH 守护进程

配置文件: `config/sshd_config`

```
Port 22222
PubkeyAuthentication yes
PasswordAuthentication no
PermitEmptyPasswords no
AllowUsers box
AllowTcpForwarding yes
GatewayPorts yes
Subsystem sftp /usr/lib/ssh/sftp-server

# ForceCommand: 所有 SSH 命令通过 debug.bridge
Match All
    ForceCommand /lzcapp/pkg/content/debug.bridge $SSH_ORIGINAL_COMMAND
```

初始化 (`config/init.sh`):
1. 生成 SSH host key (`ssh-keygen -A`)
2. 创建 `box` 用户 (空密码, docker 组成员)
3. 设置 subuid/subgid (10000:65536) 用于 Buildah
4. 生成 ed25519 密钥对
5. 配置 authorized_keys
6. 添加 `registry.lazycat.cloud` hosts 映射
7. 启动 rsync daemon
8. 启动 SSHD

### 3.2 rsync 转发

配置: `config/rsyncd.conf`

```
port = 874
uid = root
gid = root
use chroot = yes
max connections = 4
timeout = 300

[lzcapp_cache]
path = /lzcapp/run/data/app/cache/./
read only = no
```

## 6. 安全分析

### 6.1 权限过高

```yaml
ext_config:
  permissions:
    - PERM_LZC_DOCKER_ADMIN          # Docker 管理 ⇒ 容器逃逸风险
    - PERM_OTHER_APP_DATA_ADMIN      # 跨应用数据访问
    - PERM_DOCKER_CAP_SYS_ADMIN      # SYS_ADMIN ⇒ 完整系统控制
```

三个权限组合约等于宿主机 root 权限。

### 6.2 SSH 安全配置

```
PasswordAuthentication no            # 禁止密码登录
PermitRootLogin no                   # 禁止 root 登录
AllowUsers box                       # 仅允许 box 用户
ForceCommand debug.bridge            # 命令强制桥接
AllowTcpForwarding yes               # TCP 转发开启
GatewayPorts yes                     # 网关端口 (风险)
```

### 6.3 不安全镜像仓库

```go
{"insecureRegistries":["registry.lazycat.cloud"]}
```

Docker/Buildah 配置为**不验证** `registry.lazycat.cloud` 的 TLS 证书。

### 6.4 Docker Socket 权限

```bash
chmod 0777 /lzcapp/run/lzc-docker
chmod 0777 /lzcapp/run/lzc-docker/docker.sock
```

Docker socket 全局可读写 — 任何容器用户都可管理 Docker。

### 6.5 硬编码 IP

```bash
echo '172.18.0.129   registry.lazycat.cloud' >> /etc/hosts
```

此 LPK 的源码路径为 `/home/snyh/lazycat-all/lzc-cli/backend/src/`，而之前分析的 `lzc-cli` npm 包的 CLI 命令 (`lzc app install`, `lzc ssh`, 等) 高度吻合。推测关系:

```
lzc-cli (npm)                    debug.bridge (LPK)
──────────────                   ────────────────
本地开发者 CLI                    服务器端代理
CLI 发送 gRPC 请求               接受 SSH/API 请求
用户桌面端使用                    微服端运行

工作流:
lzc app install <pkg>    ──SSH──▶  debug.bridge app install
lzc ssh add-key          ──SSH──▶  debug.bridge ssh add
lzc build                ──SSH──▶  debug.bridge build
```

## 9. 总结

懒猫开发者工具是已分析的 LPK 中**权限最高、能力最强**的应用——提供 SSH 访问 → Docker 管理 → 容器构建 → 跨应用数据访问的完整开发者工作流。

### 核心能力

1. **SSH 接入**: 通过公钥认证提供安全的 SSH 访问通道
2. **Docker 管理**: 直接操作宿主机 Docker 守护进程
3. **容器构建**: 集成 Buildah，支持 Dockerfile 构建 → OCI 打包
4. **镜像仓库**: 内置 Docker Registry v2 + nginx 代理
5. **文件同步**: rsync 同步 (端口 873/874)
6. **应用管理**: LPK 安装/卸载/暂停/恢复/状态查询
7. **公钥管理**: 完整的 SSH 公钥生命周期 (申请/审批/删除)

### 风险

| 风险 | 描述 |
|---|---|
| Docker socket 全局可写 | 任何容器用户可执行 Docker 命令 |
| SYS_ADMIN capability | 具备完整的系统管理能力 |
| 不安全镜像仓库 | registry.lazycat.cloud 跳过 TLS 验证 |
| SSH 端口转发 | GatewayPorts 可能被滥用 |
| 跨应用数据访问 | 可读取其他 LPK 的数据 |
