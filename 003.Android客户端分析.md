### 3. FlyingPigeon IPC 桥接

`MainThreadServiceprovider` 继承 `com.flyingpigeon.library.ServiceContentProvider`，authority 为 `cloud.lazycat.client.serviceprovider`。

AI 浏览器 APK 中的 `LazyCatService` 通过此 Provider 通信：
- `/query/boxDomain` → 获取盒子域名
- `/query/client_cookie` → 获取认证 cookie

### 4. DLNA 投屏 (`lzc.dlna`)

通过 `RemoteMediaPlayerServiceGrpc` 实现 DLNA 媒体投屏。

### 5. Go JNI 后端 (`libgojni.so`)

与桌面端 `lzc-core.darwin` 对应的 Android 版本，Go 编译的 native 库，通过 JNI 暴露接口供 Java 调用。

## 核心发现

1. **双 APK 架构**: 客户端 APK 与 Cronite AI 浏览器分离，通过 FlyingPigeon IPC 通信
2. **Go 后端本地运行**: `libgojni.so` 是桌面 `lzc-core` 的 Android 移植版
3. **42 个 gRPC 服务接口**: 覆盖设备管理、文件传输、媒体、联系人、通知等全功能
4. **UI 层使用 WebView + 原生混合**: 主要页面通过 WebView 加载 Web 应用，原生层负责底层能力
5. **内置 WebDAV 服务器**: 支持文件远程访问
6. **DLNA 投屏**: 支持媒体推送到电视等设备

### 2. NotImplementedHandler — 空桩

**入口点**: NanoHTTPD 路由表中已映射但未实现的路由
**关键函数**: `space/heiyu/hportal/ui/webserver/route/NotImplementedHandler.java:10-17`
**调用链**:
```
HTTP GET /<未实现路由>
  → DefaultHandler.d(Map, vb.a)  // 文件:行 25
  → 返回 HTML: "The uri is mapped in the router, but no handler is specified. Status: Not implemented!"
```

## 补充分析：libgojni.so — Go JNI 后端深度解析

> 分析日期：2026-06-24 (补充)
