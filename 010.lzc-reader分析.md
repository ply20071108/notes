### 1.1 manifest.yml 完整解析

```yaml
package: manateelazycat.lzcapp.reader   # 开发者账号: manateelazycat（官方）
version: 3.6.5                            # 当前版本
name: 懒猫读书                              # 中文名
description: 简单易用的读书软件              # 描述

application:
  subdomain: reader                        # 访问子域
  routes:
    - /=http://reader-s:8000/              # 前端路由 (Next.js SPA)
    - /api/=http://reader-s:8080/api/      # API 路由 (Express/Fastify)
  multi_instance: true                     # 多实例（每个用户独立容器）
  file_handler:                            # 文件打开处理器
    mime:
      - application/pdf                    # PDF
      - application/epub+zip               # EPUB
      - x-lzc-extension/pdf                # LPK 扩展关联
      - x-lzc-extension/epub
      - x-lzc-extension/mobi               # MOBI（需转换）
      - x-lzc-extension/azw3               # AZW3（需转换）
      - x-lzc-extension/azw                # AZW（需转换）
      - x-lzc-extension/txt                # TXT（需转换）
    actions:
      open: /open?file=%u                  # 打开动作 → /open?file=<文件URL>

services:
  reader-s:                                # 服务名
    image: registry.lazycat.cloud/lazycat-reader:3.6.5  # 私有镜像
    environment:
      - NODE_ENV=production                 # Node.js 生产模式
      - PORT=8080                          # API 服务端口
      - FRONTEND_PORT=8000                 # 前端服务端口
      - UPLOAD_DIR=/app/uploads            # 上传目录
      - LOG_DIR=/app/logs                  # 日志目录
      - LOG_LEVEL=${LOG_LEVEL:-info}       # 日志级别（默认 info）
      - TZ=${TZ:-Asia/Shanghai}           # 时区（默认上海）
    binds:
      - /lzcapp/var/uploads:/app/uploads   # 持久化上传文件
      - /lzcapp/var/logs:/app/logs         # 持久化日志

locales:
  zh:
    name: 懒猫读书
    description: 懒猫读书
  en:
    name: LCMD Reader
    description: A simple and easy-to-use reading app

  enable_media_access: true                # 允许访问媒体
```

## 二、架构分析

### 2.1 服务架构：Node.js 双端口单容器

与 OpenReader-WebUI 不同（`ghcr.io/richardr1126/openreader` 单端口 `:3003`），懒猫读书采用**同一容器内双服务**架构：

| 端口 | 角色 | 技术栈 | 功能 |
|------|------|--------|------|
| `:8000` | 前端服务器 | Next.js / React SPA | 用户界面、阅读器渲染 |
| `:8080` | API 服务器 | Express / Fastify | 文件 CRUD、同步、AI 代理、格式转换 |

**分离依据：**
- 路由映射显示 `/:8000/` 和 `/api/:8080/api/` — 独立的端口绑定
- 但两者在**同一容器实例**中 (`reader-s`)，通过 Node.js 双进程或多路由共享实现

### 2.2 多实例（multi_instance: true）

每个懒猫微服用户拥有独立的 `reader-s` 容器实例绑定本地目录：

```
/lzcapp/var/uploads:/app/uploads   # 每个用户的图书目录
/lzcapp/var/logs:/app/logs         # 每个用户的日志目录
```

图书数据目录 `/lzcapp/var/uploads` 会映射到懒猫网盘的 `懒猫读书` 目录。

### 2.3 文件处理链

当用户在系统文件管理器中点击 PDF/EPUB/etc 文件时：

```
用户点击 → lazycat.files 检测 mime type → 匹配 file_handler
  → 生成 /open?file=%u URL（%u=文件资源定位符）
  → 懒猫微服系统解析 URL，打开 reader 子域下的 /open 页面
  → reader-s 容器加载文件并展示阅读器
```

### 2.4 外部队接

根据 app store 的功能描述，懒猫读书依赖的外部懒猫平台服务：

```
懒猫读书 ──→ 懒猫网盘（图书文件持久化存储）
          ──→ 懒猫 AI 算力舱（AI 翻译、OCR、DeepSeek OCR 转换、AI 听书）
          ──→ 懒猫客户端同步服务（多端进度/标注同步）
```

## 六、安全分析

### 6.1 权限模型

```
ext_config:
  enable_document_access: true   # 读取用户文档目录（含所有 PDF/EPUB 等）
  enable_media_access: true      # 读取用户媒体文件
```

应用有权访问用户的**所有文档和媒体文件**，这是实现文件浏览和打开功能所必需的。

### 6.2 容器隔离

`multi_instance: true` 确保每个用户的阅读会话和图书数据物理隔离。每个用户有独立的：

- 容器实例（独立的 Node.js 进程空间）
- `/app/uploads`（独立的图书存储目录）
- `/app/logs`（独立的日志文件）

### 6.3 安全关注点

- 镜像 `registry.lazycat.cloud/lazycat-reader:3.6.5` 是私有的，无法审计其 Dockerfile 和依赖
- 无 `content.tar` — 没有方法在安装前验证应用内容
- `enable_document_access: true` + `enable_media_access: true` 授予了广泛的文件系统访问权限

## 八、彩蛋与遗留痕迹

### 8.1 同一个开发者，多种项目名

懒猫读书在 en locale 中被命名为 **"LCMD Reader"**，与懒猫生态中的其他英文命名习惯一致（如 `lzc-cli`、`lzc-sdk`）。"LCMD" = LazyCat MicroServer Daemon/Device。

### 8.2 v3.6.5 的快速迭代

App store 页面显示 7636 次下载、13 次催更、15 条评论，评分 4.6。用户评论中 "老板代码写得又快又好，产品需求天天都有增加"（Lynx, 3/8/2026）暗示 Andy Stewart 在快速迭代该应用。

### 8.3 纯容器 LPK 模式

懒猫读书是第一个在本分析中遇到的**无 `content.tar` LPK**，表明 lazycat 平台支持两种部署模式：
1. `LPK + content.tar`（静态文件打包，如相册、网盘、智慧屏）
2. `LPK only`（纯 Docker 引用，如懒猫读书、lazycat-reader）

### 8.4 社区 vs 官方

社区版 `open-reader-lzcapp`（lazycat-contrib）和官方版 `lazycat-reader`（manateelazycat）是不同的应用。社区版使用 MIT 许可证的 OpenReader-WebUI，官方版是闭源私有应用。
