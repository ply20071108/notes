## 1. 概述

`@lazycatcloud/lzc-cli` 是懒猫微服 (Lazycat Cloud) 平台面向开发者的 CLI 工具链，提供从项目创建、构建、签名到部署的完整工作流。它同时充当盒子管理客户端，通过 gRPC 与本地客户端通信，或通过 SSH 直连远程盒子进行调试。

## 3. 核心模块分析

### 3.1 shellapi.js — gRPC 通信层

**文件**: `lib/shellapi.js` (181 行)

使用 `@grpc/grpc-js` 和 `@grpc/proto-loader` 实现与本地客户端的 gRPC 通信。

**Proto 定义**: `lib/shellapi.proto`
```
package space.heiyu.hportal.shell;
option go_package = "/shellapi";
```

**配置目录** (各平台):

| 平台 | 路径 |
|------|------|
| Linux | `~/.config/hportal-client/` |
| macOS | `~/Library/Application Support/hportal-client/` |
| Windows | `~\AppData\Roaming\hportal-client\` |

所需文件: `shellapi_addr` (连接地址) + `shellapi_cred` (凭证令牌)

**连接流程** (`lib/shellapi.js:30-75`):
1. 检测 `HTTP_PROXY` 环境变量，发出警告 (`lib/shellapi.js:32-34`)
2. 检查配置文件和凭证是否存在
3. 若配置不存在: 使用 `'fakecred'` 作为凭证 (`lib/shellapi.js:57`)，尝试从 `BOX_NAME`/`BOX_UID` 环境变量读取，否则 `process.exit(1)` (`lib/shellapi.js:62`)
4. 若配置存在: 加载 protobuf 定义，创建 `grpc.credentials.createInsecure()` 连接 (`lib/shellapi.js:69`)，设置 `lzc-shellapi-cred` 元数据头

**核心方法**:
- `queryBoxList()` — 获取盒子列表 (`lib/shellapi.js:102`)
- `queryShellCoreInfo()` — 查询 Shell Core 信息 (`lib/shellapi.js:114`)
- `setDefaultBox(boxname)` — 设置默认盒子 (`lib/shellapi.js:167`)
- `resolveClientId()` — 解析客户端 ID (`lib/shellapi.js:129`)

### 3.3 LPK 打包系统 — 完整格式规范

**文件**: `lib/lpk/` + `lib/sig/` + `lib/app/lpk_build.js` + `lib/app/lpk_build_images*.js` + `lib/package_info.js` + `lib/app/manifest_lint.js`

LPK (Lazycat Package Kit) 是懒猫微服的应用打包格式。以下基于源码还原完整规范。

---

#### 3.3.1 归档格式: ZIP / Tar 双格式

LPK 支持两种底层归档格式，通过文件头识别。

**格式检测** (`lib/utils.js:545-560`):
```
- 头 4 字节 = 0x50 4B 03 04 → ZIP
- 偏移 257 处 = "ustar" → Tar
- 否则 → 抛出异常
```

**重复的检测函数** (`lib/sig/core.js:15-31`, `lib/lpk/core.js:55-71`):
先检查文件扩展名 (`.lpk.tar` / `.tar`) → 判定 Tar；否则读前 4 字节 (`0x50 4B`) → ZIP；兜底 Tar。

**版本决定格式** (`lib/app/lpk_build.js:957`):
- LPK v1 → ZIP
- LPK v2 → Tar

**构建输出路径** (`lib/app/lpk_build.js:1016`): `{package}-v{version}.lpk`

---

#### 3.3.2 LPK v1 (旧版 ZIP) 内部结构

```
{package}-v{version}.lpk  [ZIP]
├── manifest.yml              # 所有字段（包括静态包元数据）
├── content.tar               # 可选: 内容目录压缩包（纯 tar，非 gzip）
├── icon.png                  # 可选: 应用图标 (PNG, ≤200 KiB)
├── deploy_params.yml         # 可选: 部署参数
├── compose.override.yml      # 可选: Docker Compose 覆写
├── extension.zip             # 可选: 浏览器扩展
└── ai-pod-service/           # 可选: AI pod 服务目录
```

#### 3.3.3 LPK v2 (当前 Tar) 内部结构

```
{package}-v{version}.lpk  [TAR]
├── package.yml               # 必需: 静态包元数据（v2 强制）
├── manifest.yml              # 应用运行时 manifest
├── content.tar.gz            # 可选: 内容目录压缩包（有 images 时 gzip）
├── images/                   # 可选: OCI 镜像存储
│   ├── oci-layout            # {"imageLayoutVersion": "1.0.0"}
│   ├── index.json            # OCI 索引
│   └── blobs/sha256/{64hex}  # 配置 JSON + 层 tarball
├── images.lock               # 镜像层锁文件 (YAML)
├── icon.png                  # 可选: 应用图标
├── deploy_params.yml         # 可选: 部署参数
├── compose.override.yml      # 可选: Docker Compose 覆写
├── extension.zip             # 可选: 浏览器扩展
├── ai-pod-service/           # 可选: AI pod 服务
├── exports/                  # 可选: 资源导出（资源包独有）
│   └── {kind}/{resourceId}/
└── META/                     # 可选: 签名信息
    ├── release.lock           # 文件校验清单 JSON
    ├── keys/{keyId}.pub       # Ed25519 公钥 PEM
    └── signatures/{keyId}.sig # Ed25519 签名 JSON
```

**版本判定** (`lib/lpk/core.js:262`): 检查是否存在 `images/` 目录 → v2，否则 v1


**嵌入镜像引用格式**:
manifest 中 `application.image` 或 `services.<name>.image` 可使用 `embed:alias` 语法:

- `embed:` 前缀表示引用包内 `images/` 中的镜像
- `alias` 对应 `lzc-build.yml` 中 `images.<alias>` 的名称
- 构建时自动重写添加 `@sha256:...`（`lib/app/lpk_build.js:54-105`）

```yaml
    # 或: "registry.lazycat.cloud/...@sha256:{64hex}" = 上游引用
    layers:
      - digest: "sha256:{64hex}"
        source: "embed"                       # 嵌入包内
      - digest: "sha256:{64hex}"
        source: "upstream"                    # 从上游仓库获取
```

#### 3.3.8 签名系统（META/ 目录）

**文件**: `lib/sig/core.js`（254 行）

**密钥算法**: Ed25519 (`crypto.generateKeyPairSync('ed25519')`)
- 私钥: PKCS#8 PEM `{name}.ed25519.private.pem`
- 公钥: SPKI PEM `{name}.ed25519.public.pem`

**签名流程** (`signPackageFile()`, `lib/sig/core.js:195-234`):

1. 解包 LPK 到临时目录
2. 遍历所有文件（排除 `META/`），计算 SHA-256 摘要
3. 生成 `release.lock`:
   ```json
   {
     "schema": "lazycat.lpk.release-lock/v1",
     "appid": "cloud.lazycat.app.xxx",
     "version": "0.0.1",
     "objects": [
       {"path": "manifest.yml", "digest": "sha256:{hex}", "size": 1234},
       ...
     ]
   }
   ```
4. 用 Ed25519 私钥签名 JSON 内容
5. 写入 META/ 目录:
   ```
   META/
   ├── release.lock               # 校验清单
   ├── keys/{keyId}.pub           # Ed25519 公钥
   └── signatures/{keyId}.sig     # 签名 JSON:
       {
         "schema": "lazycat.lpk.signature/v1",
         "algorithm": "ed25519",
         "key_id": "dev",
         "signed_file": "META/release.lock",
         "signature": "{base64-sig}"
       }
   ```

**已签名检测**: `hasMetaSignData()`（`lib/sig/core.js:131-141`）— 检查 `META/release.lock` 存在或 `META/signatures/*.sig` 非空

## 7. 彩蛋与遗留痕迹

### 7.1 硬编码 IPv6 DNS 服务器

**位置**: `lib/utils.js:528`

```javascript
dns.setServers(['[fc03:1136:3800::1]']);
```

懒猫微服私有网络 DNS。`fc03::/8` 是 ULA 地址空间。该调用在 `resolveDomain()` 函数中每次执行都会覆写全局 DNS 解析器，可能影响同一进程中后续的其他 DNS 查询行为。

### 7.2 heiyu.space 域名泛滥

**分布**: 全仓库 26 处引用，8 个文件

该域名 (`heiyu.space`) 是懒猫微服的基础设施域名，而非官方品牌域名 (`lazycat.cloud`)。从 proto 包名 (`space.heiyu.hportal.shell`) 到用户可见的消息文本都硬编码了该域名。`heiyu`（黑羽）推测为创始人的个人域名或早期品牌名。Proto 包命名空间为 `space.heiyu`，表明早期设计使用了 `space` 顶级域名目录。

### 7.3 heiyu-0 网络接口

**位置**: `lib/app/lpk_build.js:263`

```javascript
const result = spawn.sync('ip', ['addr', 'show', 'heiyu-0'], { encoding: 'utf-8' });
```

在非 macOS、非 Windows 系统上，通过 `ip addr show heiyu-0` 获取懒猫微服私有网络接口的 IPv6 地址，用于构建 LocalIP 反向代理配置。该接口名 (`heiyu-0`) 非标准网络接口名，属于懒猫微服的虚拟网络接口。

### 7.4 伪造 gRPC 凭证

**位置**: `lib/shellapi.js:57`

```javascript
const md = new grpc.Metadata();
md.add('lzc-shellapi-cred', 'fakecred');
this.metadata = md;
```

当 gRPC 配置文件和凭证文件不存在时，代码创建一个值为 `'fakecred'` 的虚假凭证元数据。如果环境变量 `BOX_NAME` 和 `BOX_UID` 未设置，进程将直接退出 (`process.exit(1)`)，但 `fakecred` 已经在之前被创建。这属于未完成的分支处理逻辑。

### 7.5 未实现的功能

**位置**: `lib/app/index.js:332`

```javascript
handler: async () => {
    if (!resolveBuildRemoteFromFile(process.cwd())) {
        await shellApi.init();
    }
    throw 'not yet realized';
},
```

`app log` 命令（查看应用日志）的 handler 直接抛出一个字符串异常，表明该功能尚未实现。命令定义中的参数解析（`-f`/`--follow` 参数）已完整实现，但核心逻辑缺失。

### 7.6 暴露的测试导出

**位置**: `lib/appstore/apkshell.js:108-110`

```javascript
export const __test__ = {
    buildAppNameMap,
};
```

`__test__` 导出作为内部测试接口被保留在生产代码中。虽然 `buildAppNameMap` 本身是一个无害的帮助函数，但此类约定 (`__test__` 命名) 通常是测试辅助工具的编码习惯，不应出现在发布包中。

### 7.7 废弃命令保留

**位置**: `lib/app/index.js:233`

```javascript
logger.warn('project devshell is deprecated. Use "project deploy", "project start", "project exec", "project cp", and "project log" instead.');
```

`project devshell` 命令已被废弃，但其 handler 依然完整保留（初始化、检测 rsync、构建、rsync shell），代码超过 30 行。该命令作为向后兼容保留，但可考虑在后续主要版本中移除。

### 7.8 changelog 重复条目

**文件**: `changelog.md`

v2.0.5 → v2.0.8 的 Commits 部分存在显著的内容重叠：

- v2.0.8 (28 commits) 包含 v2.0.7 的全部 18 条 + v2.0.6 的 17 条 + v2.0.5 的 12 条
- v2.0.7 (18 commits) 包含 v2.0.6 的全部 17 条 + v2.0.5 的 12 条
- v2.0.6 (17 commits) 包含 v2.0.5 的全部 12 条
- v2.0.5 (12 commits) 是最短的独立集合

模式: 每个新版本发布时将之前版本的 commit 历史整体追加，而非仅追加新增 commits。这表明 changelog 由 `@release-it/conventional-changelog` 工具自动生成，但配置存在问题——每次发布时重新包含了历史 commits。

### 7.9 insecure gRPC 连接

**位置**: `lib/shellapi.js:69`

```javascript
this.client = new pbShell.ShellCore(addr, grpc.credentials.createInsecure());
```

gRPC 连接使用 `createInsecure()`，无 TLS 加密。凭证令牌以明文方式在元数据头 `lzc-shellapi-cred` 中传输。这意味着在同一网络段上的攻击者可以嗅探到 gRPC 通信内容及认证令牌。

### 7.10 自引用依赖

**位置**: `package.json:35`

```json
"@lazycatcloud/lzc-cli": "^2.0.0"
```

`package.json` 的 `dependencies` 中包含对自身包 `@lazycatcloud/lzc-cli` 的引用。这在 npm 生态中属于反模式，会导致循环依赖问题（虽然在大多数 npm 实现中不会造成运行时错误，但会令人困惑）。
