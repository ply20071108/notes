## 自定义代码分析

### 1. LazyCatService.java (`org.chromium.chrome.browser`)

唯一新增的 Java 文件，133 行。功能：

- 通过 `127.0.0.1:31085` HTTP 代理转发请求
- 通过 `content://cloud.lazycat.client.serviceprovider` ContentProvider 与盒子通信
- 在 `<data-dir>/extensions/var/` 中管理扩展（下载、校验、清理）

**`getLazyCatBoxDomain()`**: 查询 ContentProvider 路径 `/query/boxDomain`，返回盒子域名（如 `box123.lazycat.app`）。

**`getLazyCatCookies()`**: 先获取盒子域名，再查询 `/query/client_cookie` 路径，返回盒子会话 cookie。

**扩展同步方法 `a(name, url, sha256)`**: 检查 `{name}.zip.sha256sum` 文件是否已存在且匹配，若匹配则跳过下载，否则下载 `{name}.zip` 并写入校验和文件。

### 2. ChromeTabbedActivity.java 修改

在 `onStart()` 中注入了约 50 行初始化逻辑：

```
onStart()
  │
  ├── 首次启动时初始化 LazyCatService
  │   └── 绑定 ContentProvider URI
  │
  ├── 获取盒子域名: LazyCatService.getLazyCatBoxDomain()
  │
  ├── 请求扩展列表: GET https://extensions-ai.<boxDomain>/api/list
  │   └── 返回 JSON: { "ext-name": { "url": "...", "sha256": "..." } }
  │
  ├── 下载每个扩展: LazyCatService.a(name, url, sha256)
  │   └── 存储到 <data-dir>/extensions/var/
  │
  └── 清理已删除的扩展
```

### 3. C++ 层修改 (libchrome.so)

通过 `strings` 在 native 库中发现的定制字符串：

| 字符串 | 说明 |
|---|---|
| `cookies.getLazyCatCookie` | 新增 Chrome Extension API 函数 |
| `cookies.getLazyCatBoxDomain` | 新增 Chrome Extension API 函数 |
| `getLazyCatCookies` | C++ 内部函数 |
| `getMoreExtensionsUrl` | 可能用于获取扩展列表 URL |
| `cloud.lazycat.app.ai.zip` | 硬编码的扩展包路径 |
| `org/chromium/chrome/browser/LazyCatService` | JNI 绑定的 Java 类 |
| `Lzc3ez` | 未知 token/hash |

Chromium 的 Extension API 系统被扩展了两个自定义函数，在 `chrome.cookies` 命名空间下注册：

```json
{
  "name": "getLazyCatCookie",
  "type": "function",
  "parameters": [],
  "returns_async": {
    "name": "callback",
    "parameters": [{"name": "details", "type": "string"}]
  }
},
{
  "name": "getLazyCatBoxDomain",
  "type": "function",
  "parameters": [],
  "returns_async": {
    "name": "callback",
    "parameters": [{"name": "details", "type": "string"}]
  }
}
```

运行在浏览器中的 Chrome 扩展可直接调用这两个 API 获取盒子域名和会话 cookie。

## 数据流

```
开机首次启动
    │
    ▼
ChromeTabbedActivity.onStart()
    │
    ├── 初始化 LazyCatService
    │   └── 绑定 content://cloud.lazycat.client.serviceprovider
    │
    ├── getLazyCatBoxDomain()
    │   └── ContentProvider 返回 "box123.lazycat.app"
    │
    ├── GET https://extensions-ai.box123.lazycat.app/api/list
    │   └── 返回: { "ai-summary": { "url":"/ext/ai-summary.zip", "sha256":"..." }, ... }
    │
    ├── 下载 ai-summary.zip (通过 127.0.0.1:31085 代理)
    │   └── 校验 SHA-256 → 存入 <data>/extensions/var/ai-summary.zip
    │
    └── 清理已删除的扩展
```

安装的扩展在运行时可通过新增 API 获取盒子信息：

```
chrome.cookies.getLazyCatBoxDomain()  →  "box123.lazycat.app"
chrome.cookies.getLazyCatCookie()     →  "session=abc123..."
```

APK 内部不包含任何 AI 模型或 AI 推理代码：

| 声称 | 实际实现 |
|---|---|
| "AI 浏览器" | 标准 Chromium + 运行时下载的 Chrome 扩展 |

| "AI 相册" | 不在本 APK 范围内 |
| "云端 AI" | 取决于扩展的具体实现 |

"AI 功能"完全通过运行时从盒子下载的 Chrome 扩展提供，本 APK 仅作为扩展分发渠道。

### 1. Developer Options 彩蛋 — 点击版本号 5 次

**入口点**: `关于 Chrome` 页面 → 点击应用版本号字段  
**关键函数**: `org/chromium/chrome/browser/about_settings/AboutChromeSettings.java:d0(Preference):58-101`  
**调用链**:
```

  → AboutChromeSettings.d0()   // 文件:行 58
    → this.B0 计数器递减（初始值 -1）
```

### 5. 安全评估更新

| 风险项 | 评级 | 补充说明 |
|---|---|---|
| StrictMode 禁用 | 🔴 高危 | `permitAll()` 允许 UI 线程网络 I/O，可能导致 ANR |
| Extension API cookie 泄露 | 🟡 中危 | 任何 Chrome 扩展均可调用 `getLazyCatCookie()` 获取盒子会话 |
| ContentProvider 本地 IPC | ✅ 安全 | 仅 localhost，不走网络 |
| 代理 31085 | ⚠️ 需关注 | 所有 HTTP 流量经本地代理，代理行为取决于盒子端实现 |
| 应用标签未修改 | ❌ 违规 | `android:label` 仍为 "Cromite"，用户无感知属于懒猫生态 |
| GPLv3 违规 | ❌ 确认 | 完整分析见原文许可证章节 |
