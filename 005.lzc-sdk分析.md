### 2.1 gRPC-Web 通信层

`/lzc-sdk/lang/js/` → 编译为 `dist/grpcweb.js`

```
Web App                    gRPC-Web Client               lzcAPIGateway
  │                             │                              │
  │  ─── unary() ──────────────►│                              │
  │                             │  POST /_lzc/runtime/grpc/    │
  │                             │ ───────────────────────────► │
  │                             │                              │
  │  ◄── Promise<T> ───────────│                              │
  │                             │                              │
  │  ─── invoke() ─────────────►│                              │
  │                             │  stream (Observable<T>)      │
  │  ◄── Observable<T> ────────│                              │
```

`GrpcWebImpl` 封装 `@improbable-eng/grpc-web`：

- `unary()` → Promise，一元 RPC 调用
- `invoke()` → Observable，服务端流式 RPC
- 自动处理 401 Unauthenticated → 重定向到 `/sys/login`
- `x-request-time` header 注入
- 支持 `upStreamRetryCodes` 自动重连

设备 API 鉴权流程：
```
lzcAPIGateway
  │
  ├─ POST /_lzc/deviceapi_auth_token  →  { Token, Deadline }
  │
  ├─ GrpcWebImpl(deviceApiUrl, { metadata: { lzc_dapi_auth_token } })
  │
  └─ EndDeviceProxy
```

## 5. 调用链与数据流

### 5.1 盒子侧 gRPC 通信

```
[Web App]
    │
    ├─ new lzcAPIGateway("/_lzc/runtime/grpc/")
    │     └─ 创建 GrpcWebImpl
    │     └─ bo.QuerySessionInfo() → 获取 session (uid, deviceId)
    │     └─ bo.QueryAppInfo()     → 获取 app 信息
    │
    ├─ gateway.devices.ListEndDevices({ uid })
    │     └─ 返回设备列表 [{ uniqueDeivceId, deviceApiUrl, ... }]
    │
    ├─ gateway.getDeviceProxy(udid)
    │     ├─ getDeviceURL(udid)     → 获取 deviceApiUrl
    │     ├─ POST /_lzc/deviceapi_auth_token → { Token, Deadline }
    │     ├─ 创建新 GrpcWebImpl(deviceApiUrl, lzc_dapi_auth_token)
    │     └─ return new EndDeviceProxy(rpc)
    │
    ├─ proxy.photolibrary.GetAlbums(...)
    ├─ proxy.fileHandler.List(...)
    ├─ proxy.remoteControl.SendKeyboardEvent(...)
    └─ proxy.notification.Notify({ title, body, deeplinkUrl })
```

### 5.2 客户端原生桥接

```
[WebView App]
    │
    ├─ AppCommon.LaunchApp(url, appid)
    │     ├─ isInApplication()? → 否则 window.open()
    │     ├─ isAndroidWebShell()? → android_launch_service.LaunchApp(json)
    │     ├─ isIosWebShell()? → webkit.messageHandlers.LaunchApp.postMessage()
    │     ├─ isPCWebShell()? → window.ipcRenderer.invoke("launchApp")
    │     └─ isTvOsWebShell()? → window.mipcr.LaunchApp(url, appid)
    │
    ├─ AppCommon.OpenNativeVideoPlayer(payload)
    │     └─ iOS: webkit → 原生播放器 → Observable<NativeVideoPlayerEvent>
    │
    └─ AppCommon.ScanQrCode()
          └─ iOS/Android: 原生扫码 → Promise<string>
```

> 全速模式：MANUAL, Percentage：100

### 6.11 全局 window 变量污染
**`extentions/index.d.ts`** 声明了 15+ 个全局变量：
- `window.electronAPI`, `window.webkit`, `window.lzc_bridge`
- `android`, `android_launch_service`, `android_dialog`
- `lzc_app_manager`, `lzc_status_bar`, `lzc_vibrate`
- `lzc_permission`, `lzc_bluetooth`, `lzc_link`, `lzc_share`, `lzc_toast`

### 6.12 gRPC-web 无许可证归属说明
SDK 依赖 `@improbable-eng/grpc-web`（Apache-2.0 许可证）但 SDK 本身为专有软件，无明显许可证合规说明。

### 6.13 构建影响包描述
`dist/package.json` 是构建阶段生成的副本，包含 `prepublishOnly` 脚本（clean → build → check:publish），与根 `package.json` 内容一致。
