### 6.4 gRPC-Web 服务调用

前端通过 `@improbable-eng/grpc-web` 库直接调用以下 gRPC 服务：

**BrowserOnlyProxy** (会话管理):
- `QuerySessionInfo` → SessionInfo (uid, deviceId, when)
- `QueryAppInfo` → AppInfo (boxId, appId, appDomain, httpApiProxyPath)
- `PairAllDevices` → server-stream Empty
- `QueryAPIServerInfo` → APIServerInfo (frontendVersion, backendVersion)

**EndDeviceService** (设备管理):
- `ListEndDevices` → EndDevice[]
- `RemoveEndDevice` → Empty
- `SetDeviceRemarkName` → Empty

**FileHandler** (文件操作):
- `query` → AppShortcut[] (appId, title, icon, intentAction)
- `open` → url
- `openFileManager` → Empty
- `walkDir` → file stream
- `checkWalkDir` → CheckWalkDirResponse
- `dirTree` → FileInfo[]
- `stat` → StatReply
- `syncFolder` / `copyFolder` → TaskProgress[] (WebDAV sync)
- `tarDir` → data stream
- `checkExists` → paths

### 6.5 WebSocket

- `wss://host/_lzc/ai/funasr/10095` — FunASR 语音识别 WebSocket

### 6.6 前端事件流

```
Touchpad 事件 → /api/remoteHID (WebSocket) → tvos → X11
键盘事件     → /api/remoteHID (WebSocket) → tvos → X11
鼠标事件     → POST /api/mouseMove 等 → tvos → X11
语音输入     → WS FunASR → ASR 识别 → 文字 → 键盘输入
OCR          → /ocr REST → photosearch → 文字识别 → 点击操作
VNC 渲染     → noVNC iframe → WS :8900 → VNC Proxy → tvos :5900 → 视频帧
```

### 6.7 认证

- LazyCat 设备 API token 认证: `lzc_dapi_auth_token` header
- Session/App 信息通过 `QuerySessionInfo` / `QueryAppInfo` 获取
- iOS WebView bridge: `window.webkit.messageHandlers`

## 8. 数据流与调用链

### 8.1 TV 启动流程

```
用户打开前端
  │
  ├── WebSocket → /api/remoteHID → 连接建立
  │
  ├── GET /api/status → tv-control
  │   └── tvos.Status() → docker inspect tvos 容器
  │       ├── 运行中 → 返回 IP + HDMI 状态
  │       └── 未运行 → main.tryRunTvOs()
  │           └── docker run --privileged tvos
  │               ├── sway 启动 → X11 VNC Server :5900
  │               ├── Chromium 启动 (hportal-client)
  │               └── PulseAudio 启动
  │
  ├── noVNC iframe → WS VNC → tv-control :8900
  │   └── VNC Proxy → tvos :5900
  │
  └── 页面渲染: VNC 画面 + 遥控 UI
```

### 8.2 遥控操作数据流

```
触摸板滑动:
  前端 Touchpad Event
  → POST /api/mouseMove (mouseMove handler)
  → tvos.Exec("sway input ...")
  → 或者 gRPC /SendTouchpadEvent
  → tvos → X11 指针移动

键盘按键:
  前端 KeyEvent
  → WebSocket /api/remoteHID (remoteHid handler)
  → gRPC /SendKeyboardEvent
  → tvos → X11 按键事件

蓝牙音频:
  前端蓝牙配对
  → POST /api/... → gRPC /BleStartScan
  → tvos → Bluetooh 扫描
  → gRPC /BleScanDevices → 返回设备列表
  → gRPC /BleConnectDevice → 连接
  → PulseAudio SetDefaultSink → 切换音频输出

音量控制:
  前端音量按键
  → gRPC /IncreaseVolume /DecreaseVolume
  → tvos → pactl set-sink-volume

浏览器操作:
  前端 Chrome Actions
  → gRPC /BrowserAction (back/forward/refresh/newTab)
  → tvos.Exec("firefox --new-tab URL")
```

### 8.3 VNC 渲染数据流

```
tvos 容器内:
  sway (Wayland) → X11 (Xvnc)
  → VNC Server (:5900) → RFB 协议
    │
    ▼ TCP 5900
tv-control (VNC Proxy):
  go-vncproxy v1.1.0
  → WebSocket 隧道 (RFC 6455)
    │
    ▼ WebSocket
前端 (noVNC 1.6.0):
  RFB.js 解码
  → HTML5 Canvas 渲染
  → 键盘/鼠标事件 → WebSocket → VNC Proxy → X11
```

### 8.4 HDMI 检测

```
电视或显示器插入/拔出
  → udev / sysfs 事件
  → main.listenAIRunning.func2 (定时轮询)
  → tvos.IsHDMIConnected()
  → docker exec tvos "echo $HDMI" 或读取 HDMI 状态
  → 返回 true/false
  → 前端显示 "HDMI未连接" / "连接成功"
```

## 11. 彩蛋与遗留痕迹

### 11.1 开发者身份: linakesi

与远程微信和懒猫相册为同一开发者。

依赖路径: `gitee.com/linakesi/lzc-sdk`, `gitee.com/linakesi/lzc-baseos-protos`, `gitee.com/linakesi/remotesocks`

### 11.2 拼写错误: "Seletor"

- **位置**: gRPC 方法名: `/cloud.lazycat.apis.localdevice.DialogManager/OpenFileSeletor`
- **正确拼写**: `Selector`
- **影响**: 该错误在 proto 定义和生成的代码中均已固化

### 11.3 拼写错误: "Natual"

- **位置**: `languages/en/translation.json:126`
- **内容**: `"text_scroll_natural": "Natual scroll"` (应为 Natural)
- **影响**: 用户界面上的显示错误

### 11.4 残留 TODO 文件

- **位置**: proto 文件名: `baseos/todo_remove_osupgrade.proto`
- **说明**: 一个带有 "TODO" 字样的 protobuf 文件未被清理，包含在构建产物中

### 11.5 DebugTest RPC

- **位置**: `cloud.lazycat.apis.localdevice.DebugTestReply`
- **说明**: 存在调试用 RPC 方法，正式发布未移除

### 11.6 FIPS 140-3 合规代码

二进制中包含大量 FIPS 合规检测字符串:
```
"FIPS 140-3 self-test passed:"
"crypto/rsa: use of keys smaller than 2048 bits is not allowed in FIPS 140-only mode"
"crypto/rsa: only crypto/rand.Reader is allowed in FIPS 140-only mode"
```
说明开发者/CI 环境启用了 Go 的 FIPS 构建模式。

### 11.7 构建路径泄露

GoReSym 输出的 ModuleMeta 由于 stripped 1.24+ 无法恢复构建路径，但前端 JS 文件包含 hash 版本号 (如 `index-DGLB7x46.js`)，说明是 Vite 标准构建。

### 11.8 Electron/Desktop 引用残留

前端代码中包含 `window.lzc_bridge` 和 Electron API 引用，以及 `SettingPC`/`SettingDesktop` 组件，表明该应用亦可作为 Electron 桌面应用运行。

### 11.9 品牌混淆: "LCMD"

英文翻译中多处使用 "LCMD" 而非 "LazyCat":
- "Please connect the LCMD MicroServer"
- "Please proceed LCMD Appstore"
说明可能存在品牌迁移或品牌重塑。

### 11.10 pullse → pulse 拼写

- **位置**: i18n keys 命名: `tv_pulse_audio_*` 系列
- **说明**: "pulse" 误写为 "pulse"

### 11.11 grpc-go 版本字符串

```
grpc-go/1.67.1
```
GoReSym 验证 gRPC 版本为 1.67.1。

### 11.12 未使用的 Fullscreen API

前端代码存在 `Fullscreen API` 调用残留，可能曾计划支持全屏模式。

### 11.13 前端 JS 中的 "META_KEY_X_LZCAPI_REALUID"

二进制字符串中显示 `META_KEY_X_LZCAPI_REALUID` — 一个自定义 HTTP header，用于传递真实用户 ID。
