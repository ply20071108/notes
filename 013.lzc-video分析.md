              │ HLS stream     │  └──────────────────┘

### 关键特征

- **纯前端 SPA**: 无任何后端代码，所有逻辑在浏览器中执行
- **GPU 加速**: `gpu_accel: true` — 利用硬件视频解码
- **MIME 处理器**: 处理 20+ 视频格式，通过 `/open?url=%u` 入口打开
- **PWA**: Service Worker (Workbox 6.5.3) + manifest.webmanifest
- **媒体访问**: `enable_media_access: true` — 可直接访问媒体文件

### HLS 流媒体

视频文件通过 lazycatcloud 媒体服务器转码为 HLS 流：

```
原始视频文件
    │
    ▼
lazycatcloud 媒体服务器
    │
    ▼
HLS 分段 (TS/fMP4)
    │
    ▼
URL 格式:
  https://video.{domain}.heiyu.space/_lzc/media/hls{filename}/quality-720.m3u8
  https://video.{domain}.heiyu.space/_lzc/media/hls{filename}/master.m3u8

hls.js 处理:
  - 加载 master.m3u8 获取可用码率
  - ABR 自动选择最优码率
  - 支持手动切换清晰度
  - 内置错误恢复机制
```

### 字幕渲染 — SubtitlesOctopus

使用 libass 的 WebAssembly 编译版本实现高质量字幕渲染：

```
字幕来源:
├── 内置字幕 (HLS 流内)
│   └── SubtitleTrackController 提取
├── 外挂字幕 (ASS/SSA 文件)
│   └── subtitleInfoUrl 获取
└── 隐藏字幕
    └── CEA-608/708 解码

渲染流程:
  字幕数据 → SubtitlesOctopus → Web Worker
      → libass WASM 渲染 → Canvas overlay → 视频上方叠加
```

SubtitlesOctopus 内部组件:
- brotli 解压 (Google 实现)
- expat XML 解析器
- libass WASM 二进制 (~2.2MB)
- Worker 线程渲染 (主/legacy 两个版本)

### 投屏实现

```
DLNA (UPnP) 投屏:
  gRPC 服务: lzc.dlna.RemoteMediaPlayerService
  ├── 设备发现 (SSDP)
  ├── 媒体 URL 推送
  └── 远程控制 (播放/暂停/进度)

AirPlay:
  <video> disableRemotePlayback 属性管理
  通过浏览器原生 AirPlay 支持

投屏状态机:
  ┌────────┐   发现设备   ┌────────┐   推送 URL   ┌────────┐
  │  空闲   │───────────▶│ 设备选择 │────────────▶│ 投屏中  │
  └────────┘            └────────┘            └────────┘
                                                    │
                                              返回播放
                                                    │
                                                    ▼
                                                ┌────────┐
                                                │ 本地播放 │
                                                └────────┘
```

### 错误处理

| 错误类型 | 用户提示 | 处理策略 |
|---|---|---|
| 解码失败 | 支持 mp4, m3u8, mkv | 提示格式不支持 |
| 加密视频 | 没有密码解密 | 提示加密（不支持解密） |
| 文件不存在 | 没有访问权限或文件已不存在 | 提示用户检查文件 |
| 网络错误 | 媒体下载中途失败 | hls.js 自动重试 |
| 请求中断 | 请求资源终断 | 需用户手动重新播放 |
| 未知错误 | 未知错误 | 通用提示 |

使用 `lzc:subtitle-size-level` localStorage key 存储用户字幕大小偏好 — 遵循 `lzc:` 前缀惯例。

index.531ca89a.js (1MB) 包含完整的 RxJS 库，这表明 Pinia store 或某些服务使用了 RxJS Observable 模式（可能是用于处理 HLS 事件流或 DLNA 设备发现）。

## 12. 总结

懒猫视频播放器是已分析的 LPK 中**最简单的应用** — 一个纯粹的**前端视频播放器 SPA**，无后端代码。其核心价值在于：

1. **HLS 转码代理**: 通过 lazycatcloud 媒体服务器将任意视频文件转码为 HLS 流
2. **高质量字幕渲染**: libass WASM (SubtitlesOctopus) — 支持 ASS/SSA 特效字幕
3. **多协议投屏**: DLNA (UPnP) + AirPlay
4. **PWA 离线能力**: Workbox 预缓存所有静态资源
5. **20+ 视频格式支持**: 通过 MIME handler 注册

### 与其他 LPK 的对比

| LPK | 类型 | 后端 | 复杂度 |
|---|---|---|---|
| 懒猫相册 | 全栈 | Node.js | 高 |
| 懒猫网盘 | 全栈 | Node.js + OWS | 高 |
| 懒猫微信 | 容器 | CDE sidecar | 高 |
| 懒猫智慧屏 | 全栈 | Go VNC + Chromium | 高 |
| 懒猫阅读器 | 容器 | Node.js | 中 |
| 懒猫音乐 | 全栈 | Node.js + SQLite | 高 |
| **视频播放器** | **纯前端** | **无** | **低** |
| LightOS | 系统 | Go 二进制 | 极高 |

### 局限性

1. **依赖外部媒体服务器**: 所有视频必须经过 lazycatcloud 的 HLS 转码服务
2. **不支持加密视频**: 错误提示明确表示 "没有密码解密"
3. **仅支持 HLS 协议**: 未集成 DASH/WebRTC 等其他流媒体协议
4. **无后端转码能力**: 所有转码由平台层完成，LPK 自身无 ffmpeg
